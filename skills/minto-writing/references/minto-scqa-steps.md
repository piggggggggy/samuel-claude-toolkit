# Six Steps for Building an Intro (SCQA)

Minto's step-by-step process for writing an intro. Use when building SCQA from scratch or when C is proving difficult to find.

---

## Core Premise

Minto's design order is not S → C → Q → A.
It is **A (or Q) → S → C → Q verification**.
Start with what you want to say, then find S and C to support it.

---

## The Six Steps

### Step 1 — The Topic Box
- Write down the topic you want to convey (candidate for Answer)
- If unclear, move to Step 2

### Step 2 — Determine the Question (Q)
- Picture the reader concretely
- What question do you want this piece to answer for them?
- If the question is clear, write it down
- If not, skip to Step 4

### Step 3 — Write the Answer (A)
- If you know the answer, write it down
- If not, note "can be answered" and continue

### Step 4 — Clarify the Situation (S)
- Start with what the reader can easily accept
- Things that are uncontroversial, that the reader already knows, or can quickly verify and agree with
- Question: "What does the reader take for granted at this moment?"

### Step 5 — Find the Complication (C)
- Imagine the reader nodding along to S, then asking "I know, so what?"
- Find what caused that question to arise:
  - Something went wrong
  - A problem has emerged
  - There's a logical inconsistency
  - A new layer has been added that disrupts the stability of S
- Question: "Despite S being stable, why does tension exist right now?"

### Step 6 — Verify Q and C Alignment
- Does C immediately provoke Q?
- If not:
  - Reword Q to fit C, or
  - Check whether C itself is wrong
- Test: Does a reader who encounters C naturally arrive at Q?

---

## Useful Hints

The hardest cases for finding C are when S is simply repeated or negated.

Good C adds a dimension to S that wasn't there before. Reading S deeply will usually reveal C latent within it.

---

## Supporting Tool — Problem Definition R1/R2

Use when C or Q remains unclear even after working through the six steps.

- **R1 (Starting point)**: Current state — the unwanted result, what's happening now
- **R2 (Destination)**: Desired state — the goal, the ideal outcome
- **Gap**: The difference between R1 and R2 → this is what creates C and narrows Q

**How to use**:

```
R1: What is actually happening right now?
R2: What should be happening, or what is the desired state?
Gap: What is the difference between the two?
```

Once the gap is clear, C becomes "we're at R1, not R2," and Q naturally narrows to "how do we close the gap?"

**Example**:
- R1: New user retention keeps falling
- R2: 7-day retention above 40%
- Gap: Currently 22%, 18 points below target
- C: More than half of new users are dropping off within their first week
- Q: What is driving the churn, and how do we stop it?
